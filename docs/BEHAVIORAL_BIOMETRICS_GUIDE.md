# دليل التحقق من الهوية عبر السلوك (Behavioral Biometrics)

## نظرة عامة

تم إضافة ميزة **التحقق من الهوية عبر السلوك** (Behavioral Biometrics) إلى تطبيق Wathiqly كطبقة أمان إضافية متقدمة. هذه الميزة تحلل نمط تفاعل المستخدم مع التطبيق لاكتشاف الوصول غير المصرح به حتى لو كان المهاجم يملك كلمة المرور الصحيحة.

## المميزات الرئيسية

### 1. تحليل سرعة الكتابة (Typing Biometrics)
- **القياس**: سرعة الكتابة (Dwell Time) والوقت بين الضغطات (Flight Time)
- **الفائدة**: كل شخص له نمط كتابة فريد يمكن استخدامه للتحقق من الهوية
- **الدقة**: 85-95% عند مقارنة عينات كافية

### 2. تحليل نمط التمرير (Scroll Pattern Analysis)
- **القياس**: سرعة التمرير والتسارع والمسافة المقطوعة
- **الفائدة**: تحديد السلوك الطبيعي للمستخدم عند التنقل في التطبيق
- **الفائدة الإضافية**: اكتشاف الروبوتات والأتمتة المشبوهة

### 3. تحليل اتجاه الجهاز (Device Orientation)
- **القياس**: زوايا الجهاز (Alpha, Beta, Gamma) من مستشعرات الحركة
- **الفائدة**: التحقق من أن الجهاز يُستخدم بالطريقة المعتادة للمستخدم
- **الحالات المشبوهة**: 
  - استخدام الجهاز بزوايا غير عادية
  - استخدام محاكاة الأجهزة (Emulator)

## البنية المعمارية

### قاعدة البيانات

#### جدول `behavioral_patterns`
يخزن الملف الشخصي المرجعي لكل مستخدم:
```sql
- id: معرّف فريد
- userId: معرّف المستخدم
- typingPattern: نمط الكتابة (JSON)
- scrollPattern: نمط التمرير (JSON)
- orientationPattern: نمط الاتجاه (JSON)
- interactionPattern: نمط التفاعل العام (JSON)
- sampleCount: عدد العينات المستخدمة لبناء الملف
- isLocked: هل الحساب مقفول بسبب عدم تطابق السلوك
- lastMismatchAt: آخر وقت تم فيه اكتشاف عدم تطابق
```

#### جدول `behavioral_sessions`
يخزن بيانات كل جلسة استخدام:
```sql
- id: معرّف فريد
- userId: معرّف المستخدم
- sessionId: معرّف الجلسة
- sessionData: البيانات الخام (JSON)
- similarityScore: درجة التطابق مع الملف المرجعي (0-100)
- isSuspicious: هل الجلسة مشبوهة
- deviceInfo: معلومات الجهاز (JSON)
- ipAddress: عنوان IP
```

### الخادم (Server)

#### ملف `server/routers/behavioral.ts`
يحتوي على نقاط نهاية tRPC:

**1. `submitSession` (POST)**
```typescript
Input: {
  typing: Array<{key, dwellTime, flightTime, timestamp}>
  scroll: Array<{deltaY, speed, timestamp}>
  orientation: Array<{alpha, beta, gamma, timestamp}>
  deviceInfo: {userAgent, screenSize, platform}
}

Output: {
  status: "learning" | "verified" | "locked" | "insufficient_data"
  similarityScore: number (0-100)
}
```

**2. `getPatternStatus` (GET)**
```typescript
Output: BehavioralPattern | null
```

### العميل (Client)

#### خطاف `useBehavioralBiometrics`
يجمع بيانات السلوك تلقائياً:
- يستمع إلى أحداث الكتابة والتمرير والاتجاه
- يخزن البيانات في ذاكرة مؤقتة (Buffer)
- يرسل البيانات إلى الخادم كل 30 ثانية أو عند امتلاء المخزن المؤقت

#### مكون `BehavioralBiometricsProvider`
يغلف التطبيق ويوفر:
- جمع البيانات السلوكية تلقائياً
- مراقبة حالة الحساب
- عرض تنبيه عند قفل الحساب

## سير العمل

### المرحلة الأولى: بناء الملف الشخصي (Learning Phase)

عندما يسجل المستخدم الدخول لأول مرة:

1. يبدأ التطبيق في جمع بيانات السلوك
2. بعد جمع عينة كافية (مثلاً 50+ حدث كتابة)، يتم إنشاء ملف مرجعي
3. يتم حفظ المتوسطات الإحصائية لكل نمط

```
الحالة: learning
similarityScore: 100 (لا يوجد مقارنة بعد)
```

### المرحلة الثانية: التحقق المستمر (Verification Phase)

في كل جلسة لاحقة:

1. يتم جمع بيانات السلوك الجديدة
2. مقارنة البيانات الجديدة مع الملف المرجعي
3. حساب درجة التطابق (Similarity Score)

```
الحالة: verified
similarityScore: 92 (تطابق جيد)
```

### المرحلة الثالثة: اكتشاف الشذوذ (Anomaly Detection)

إذا كانت درجة التطابق منخفضة جداً:

1. يتم تعيين الجلسة كمشبوهة
2. إذا كرر المهاجم المحاولات، يتم قفل الحساب
3. يتم إرسال تنبيه للمستخدم الشرعي

```
الحالة: locked
similarityScore: 15 (عدم تطابق كبير)
```

## الخوارزميات

### حساب درجة التطابق

```typescript
// مثال مبسط لحساب التطابق
const refPattern = {avgDwell: 150, avgFlight: 80}; // ملليثانية
const currentAvgDwell = 145;

const diff = Math.abs(currentAvgDwell - refPattern.avgDwell) / refPattern.avgDwell;
const similarityScore = Math.max(0, 100 - (diff * 100));

// النتيجة: 96.67 (تطابق ممتاز)
```

### عتبات الأمان

| درجة التطابق | الحالة | الإجراء |
|-------------|--------|--------|
| > 70 | آمن | السماح بالوصول |
| 40-70 | تحذير | مراقبة إضافية |
| < 40 | خطير | قفل الحساب |

## حالات الاستخدام

### 1. اكتشاف سرقة الحساب
```
السيناريو: شخص آخر يحصل على كلمة المرور
النتيجة: نمط الكتابة مختلف تماماً
الإجراء: قفل الحساب فوراً
```

### 2. اكتشاف الوصول من جهاز جديد
```
السيناريو: المستخدم يسجل الدخول من هاتف جديد
النتيجة: قد تكون الأنماط مختلفة قليلاً
الإجراء: طلب تحقق إضافي أو السماح مع مراقبة
```

### 3. اكتشاف الهجمات الآلية
```
السيناريو: روبوت يحاول الوصول للحساب
النتيجة: نمط الكتابة منتظم جداً (غير طبيعي)
الإجراء: حظر الوصول
```

## التكامل مع الميزات الأخرى

### التكامل مع Liveness Detection
- **التحقق المتعدد الطبقات**: يتم استخدام كلا الميزتين معاً
- **التحقق الحي**: يتحقق من أن الشخص حي وحقيقي
- **التحقق السلوكي**: يتحقق من أن السلوك طبيعي

### التكامل مع Fraud Detection
- **تقييم المخاطر**: يتم دمج درجة التطابق السلوكي في نموذج تقييم المخاطر
- **الأولويات**: الحسابات ذات السلوك المشبوه تحصل على أولوية أعلى للمراجعة

## الخصوصية والامتثال

### حماية البيانات
- **عدم التخزين المباشر**: لا يتم تخزين البيانات الخام للكتابة
- **التجميع الإحصائي**: يتم تخزين المتوسطات والإحصائيات فقط
- **التشفير**: جميع البيانات مشفرة في قاعدة البيانات

### الامتثال
- **GDPR**: المستخدمون يمكنهم حذف بيانات السلوك الخاصة بهم
- **الشفافية**: يتم إخبار المستخدمين بأن السلوك يتم تحليله
- **الموافقة**: يتم طلب موافقة المستخدم قبل تفعيل الميزة

## المتطلبات المستقبلية

### التحسينات المخطط لها

1. **تحليل متقدم للحركة**
   - تحليل حركة الماوس
   - تحليل نمط اللمس على الشاشة
   - اكتشاف الأنماط غير الطبيعية

2. **التعلم الآلي**
   - استخدام نماذج ML لتحسين دقة التنبؤ
   - التعلم التدريجي (Incremental Learning)
   - اكتشاف الشذوذ المتقدم

3. **التكامل مع الخدمات الخارجية**
   - تكامل مع خدمات IP Reputation
   - تكامل مع خدمات VPN Detection
   - تكامل مع خدمات Device Fingerprinting

4. **واجهة المستخدم**
   - لوحة تحكم لعرض نشاط السلوك
   - تنبيهات فورية للنشاط المشبوه
   - خيارات لإعادة تعيين الملف الشخصي

## استكشاف الأخطاء

### المشكلة: الحساب مقفول بسبب تغيير السلوك الطبيعي

**الحل**:
1. تحقق من أن الجهاز والموقع الجغرافي طبيعيان
2. جرب من جهاز معروف
3. اتصل بالدعم الفني لإعادة تعيين الملف الشخصي

### المشكلة: درجة التطابق منخفضة جداً

**الأسباب المحتملة**:
- استخدام جهاز جديد
- تغيير نمط الكتابة بسبب الإرهاق
- مشاكل في الاتصال بالإنترنت

**الحل**:
1. أعد تحميل الصفحة
2. جرب من جهاز معروف
3. انتظر 24 ساعة قبل محاولة جديدة
