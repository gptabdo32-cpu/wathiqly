# نظام الضمان الذكي (Smart Escrow System) - منصة وثقلي

تم تصميم هذا النظام لتعزيز الثقة والشفافية في المعاملات التجارية للمنتجات المادية والخدمات، من خلال دمج تقنيات حديثة تضمن حقوق جميع الأطراف.

## 1. المكونات الرئيسية

### أ. نظام المراحل (Milestones System)
- **الهدف**: تقسيم الخدمات الكبيرة إلى مراحل دفع صغيرة.
- **الآلية**: يتم تحرير جزء من المبلغ تلقائياً عند إنجاز كل مرحلة.
- **التحقق**: يدعم التحقق اليدوي أو التلقائي (مثل رفع الكود على GitHub).

### ب. تكامل إنترنت الأشياء (IoT Integration)
- **الهدف**: تتبع حالة وموقع المنتجات المادية عالية القيمة.
- **الأجهزة المدعومة**:
  - **GPS Trackers**: لتأكيد الوصول إلى الموقع المحدد.
  - **Temperature Sensors**: للمنتجات الحساسة للحرارة.
  - **Impact Sensors**: للكشف عن سوء التداول أثناء الشحن.
- **الأتمتة**: يمكن برمجة النظام لتحرير الأموال فور وصول المنتج للمشتري (Geofencing).

### ج. سجل الشفافية (Blockchain Transparency)
- **الهدف**: توفير سجل غير قابل للتغيير لجميع حركات الصفقة.
- **التقنية**: تسجيل الأحداث الرئيسية (تمويل، إنجاز مرحلة، تحرير أموال) على شبكة البلوك تشين (مثل Polygon).
- **الفائدة**: مرجع قانوني وتقني موثوق في حالة النزاعات.

## 2. الهيكل التقني (Technical Architecture)

### قاعدة البيانات (Database Schema)
- `escrowMilestones`: تخزين مراحل العمل، المبالغ، وحالة التحقق.
- `iotDevices`: ربط الأجهزة المادية بالصفقات وتخزين آخر القراءات.
- `blockchainLogs`: تخزين هاش المعاملات (Transaction Hashes) للرجوع إليها.

### واجهات البرمجة (API Endpoints)
- `smartEscrow.addMilestones`: إضافة مراحل للصفقة.
- `smartEscrow.registerDevice`: ربط جهاز IoT بمنتج.
- `smartEscrow.reportIotData`: استقبال البيانات من الأجهزة (Webhook).
- `smartEscrow.getBlockchainLogs`: عرض سجل الشفافية للمستخدم.

## 3. المعايير الأمنية والهندسية
- **التشفير**: جميع بيانات IoT الحساسة مشفرة.
- **التحقق**: نظام tRPC يضمن صحة البيانات (Type-safety) في جميع مراحل التواصل.
- **اللامركزية**: استخدام البلوك تشين يمنع التلاعب في سجلات الصفقات.
