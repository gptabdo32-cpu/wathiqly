# نظام التقييم الاجتماعي الموثوق (Trusted Social Rating System) - وثيقة التصميم

## 1. نظرة عامة
يهدف هذا النظام إلى بناء بيئة تجارية آمنة وموثوقة في منصة "واثقلي" من خلال حساب "درجة ثقة اجتماعية" (Social Trust Score) لكل مستخدم بناءً على سلوكه الفعلي ومعاملاته الموثقة.

## 2. المكونات الرئيسية

### أ. درجة الثقة الديناميكية (Dynamic Trust Score)
يتم حساب الدرجة (من 0 إلى 100) بناءً على العوامل التالية:
| العامل | الوزن | الوصف |
| :--- | :--- | :--- |
| **المعاملات الناجحة** | 40% | عدد وقيمة المعاملات التي تمت بدون نزاعات. |
| **التحقق من الهوية (KYC)** | 20% | حالة التحقق (غير موثق، هاتف، هوية وطنية، بصمة وجه). |
| **تاريخ النزاعات** | -30% | خصم نقاط عند فتح نزاعات يثبت فيها خطأ المستخدم. |
| **سرعة الاستجابة** | 10% | متوسط الوقت المستغرق للرد على الرسائل وإتمام الخطوات. |
| **دقة الوصف** | 30% | تقييمات المشترين/البائعين المحددة لجودة المنتج/الخدمة. |

### ب. التقييمات الموثقة (Verified Reviews)
*   **القيد الصارم:** لا يمكن إضافة تقييم إلا بعد تغيير حالة المعاملة (Escrow) إلى `completed`.
*   **الاتجاهين:** يمكن للبائع تقييم المشتري والعكس.
*   **التحقق الآلي:** النظام يمنع التقييمات المتكررة من نفس الحسابات لمنع التلاعب (Sybil Attack).

### ج. الشارات والامتيازات (Badges & Privileges)
| الشارة | المتطلبات | الامتيازات |
| :--- | :--- | :--- |
| **بائع موثوق** | درجة > 80 + 10 معاملات | رسوم وساطة أقل (2% بدلاً من 2.5%) |
| **مشتري ممتاز** | درجة > 85 + 5 معاملات | حدود سحب وإيداع أعلى |
| **عضو ذهبي** | درجة > 95 + KYC كامل | دعم فني ذو أولوية + شارة ذهبية |

### د. التكامل الاجتماعي (Social Integration)
*   توليد "بطاقة ثقة رقمية" (Digital Trust Card) قابلة للمشاركة.
*   رابط فريد لملف الثقة (Trust Profile) يعرض الإحصائيات والشارات.

## 3. التصميم التقني

### نماذج قاعدة البيانات الجديدة
1.  `user_trust_scores`: لتخزين الدرجة الحالية وتاريخ التغييرات.
2.  `user_badges`: لتخزين الشارات الممنوحة لكل مستخدم.
3.  `verified_reviews`: (توسيع الجدول الحالي) لإضافة معايير تقييم تفصيلية (الدقة، السرعة، التواصل).

### الخوارزمية (Trust Algorithm)
سيتم تنفيذ وظيفة دورية (أو عند كل معاملة) لتحديث الدرجة:
`Score = (Success_Rate * 40) + (KYC_Level * 20) + (Avg_Rating * 30) + (Response_Factor * 10) - (Dispute_Penalty)`

## 4. المعايير الأمنية
*   **منع التلاعب:** تشفير بيانات التقييم الحساسة.
*   **النزاهة:** سجل مراجعة (Audit Log) لكل تغيير في درجة الثقة.
*   **الخصوصية:** لا يتم عرض تفاصيل المعاملات الخاصة، فقط الإحصائيات العامة.
