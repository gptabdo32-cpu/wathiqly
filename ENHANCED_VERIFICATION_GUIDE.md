# دليل نظام التحقق من الهوية المحسّن - منصة واثقلي (Wathiqly)

## نظرة عامة

تم تحسين نظام التحقق من الهوية على منصة Wathiqly بشكل كبير لتوفير تجربة أكثر أماناً وموثوقية. يتضمن النظام الجديد:

1. **خدمة OCR حقيقية** - استخراج بيانات دقيقة من بطاقات الهوية باستخدام Gemini Vision API
2. **مطابقة وجه متقدمة** - مقارنة دقيقة بين صور الهوية والسيلفي مع كشف الحياة
3. **نظام إشعارات شامل** - إخطارات فورية لكل مرحلة من مراحل التحقق
4. **كشف الاحتيال** - نظام متقدم لكشف المحاولات المريبة والاحتيالية

---

## المميزات الجديدة

### 1. خدمة OCR الحقيقية

#### الملف: `server/_core/ocr.ts`

**المميزات:**
- استخراج دقيق للبيانات من بطاقات الهوية
- دعم اللغات المتعددة (العربية والإنجليزية)
- درجة ثقة عالية (90%+) للبيانات المستخرجة
- معالجة أخطاء شاملة مع رسائل واضحة

**البيانات المستخرجة:**
```typescript
{
  fullName: string;           // الاسم الكامل
  nationalIdNumber: string;   // رقم الهوية الوطنية
  dateOfBirth?: string;       // تاريخ الميلاد
  gender?: string;            // الجنس
  nationality?: string;       // الجنسية
  expiryDate?: string;        // تاريخ انتهاء الصلاحية
  confidence: number;         // درجة الثقة (0-100)
  rawText: string;            // النص الخام المستخرج
}
```

**الاستخدام:**
```typescript
import { extractIdDataFromImage } from '../_core/ocr';

const extractedData = await extractIdDataFromImage(imageUrl);
console.log(`Extracted: ${extractedData.fullName} (${extractedData.confidence}% confidence)`);
```

### 2. مطابقة الوجه المتقدمة

#### الملف: `server/_core/faceRecognition.ts`

**المميزات:**
- مقارنة دقيقة بين وجهي الهوية والسيلفي
- كشف الحياة (Liveness Detection) للتحقق من أن السيلفي حي
- تحليل تفصيلي لملامح الوجه
- حساب درجة المخاطر (Risk Score)
- كشف الصور المزيفة والمعدلة (Deepfakes)

**نتائج المقارنة:**
```typescript
{
  matchScore: number;         // درجة التطابق (0-100)
  isMatch: boolean;           // هل الوجوه متطابقة (>= 90%)
  confidence: number;         // درجة الثقة (0-100)
  livenessScore?: number;     // درجة كشف الحياة (0-100)
  isLive?: boolean;           // هل الصورة حية (>= 80%)
  details: {
    facialFeaturesMatch: boolean;
    skinToneMatch: boolean;
    faceShapeMatch: boolean;
    eyePositionMatch: boolean;
    noseShapeMatch: boolean;
    mouthShapeMatch: boolean;
  };
  warnings: string[];         // التحذيرات المحتملة
}
```

**معايير الموافقة:**
- درجة التطابق >= 90%
- درجة الثقة >= 70%
- درجة كشف الحياة >= 80% (إن وجدت)
- تطابق 3 من 4 ملامح رئيسية على الأقل

### 3. نظام الإشعارات الشامل

#### الملف: `server/_core/verificationNotifications.ts`

**أنواع الإشعارات:**

| النوع | الوصف | الحالة |
|------|-------|--------|
| `verification_started` | بدء عملية التحقق | معلومة |
| `verification_success` | نجاح خطوة من الخطوات | نجاح |
| `verification_failed` | فشل خطوة من الخطوات | فشل |
| `verification_warning` | تحذير أثناء التحقق | تحذير |
| `verification_rejected` | رفض من الإدارة | رفض |

**الإشعارات المتاحة:**

```typescript
// عند نجاح التحقق من الهاتف
await notifyPhoneVerificationSuccess(userId, phone);

// عند فشل التحقق من الهاتف
await notifyPhoneVerificationFailed(userId, reason);

// عند نجاح رفع الهوية
await notifyIdUploadSuccess(userId, confidence);

// عند فشل رفع الهوية
await notifyIdUploadFailed(userId, reason);

// عند نجاح مطابقة الوجه
await notifyFaceMatchSuccess(userId, matchScore, livenessScore);

// عند فشل مطابقة الوجه
await notifyFaceMatchFailed(userId, reason, warnings);

// عند وضع الطلب تحت المراجعة
await notifyVerificationFlagged(userId, reason);

// عند الموافقة من الإدارة
await notifyVerificationApproved(userId);

// عند الرفض من الإدارة
await notifyVerificationRejected(userId, reason);
```

### 4. نظام كشف الاحتيال

#### الملف: `server/_core/fraudDetection.ts`

**الفحوصات المتاحة:**

1. **محاولات التحقق المتكررة**
   - أكثر من 3 محاولات في 24 ساعة = تحذير
   - أكثر من 10 محاولات من نفس IP في 7 أيام = تحذير

2. **الهويات الوطنية المكررة**
   - نفس الهوية لأكثر من حساب = حرج
   - نفس الهوية في سجل التحقق = رفض

3. **الصور المريبة**
   - درجة تطابق < 50% = مريب
   - درجة كشف الحياة < 50% = مريب
   - علامات تحرير أو تزييف = مريب

4. **نشاط IP المريب**
   - عناوين IP متعددة من نفس المستخدم
   - عناوين IP بنشاط عالي جداً

5. **حسابات جديدة**
   - محاولة التحقق في أقل من ساعة من إنشاء الحساب = معلومة

**درجات المخاطر:**

| المستوى | النطاق | الإجراء |
|--------|--------|---------|
| منخفض | 0-30 | موافقة تلقائية |
| متوسط | 30-60 | مراجعة يدوية |
| عالي | 60-80 | مراجعة يدوية مكثفة |
| حرج | 80-100 | رفض أو تعليق الحساب |

**الاستخدام:**

```typescript
import { assessFraudRisk, isSuspiciousFaceImage } from '../_core/fraudDetection';

// تقييم مخاطر الاحتيال
const assessment = await assessFraudRisk(userId, ipAddress, userAgent);
if (assessment.shouldReview) {
  console.log('Fraud risk detected:', assessment.flags);
}

// فحص الصور المريبة
const isSuspicious = isSuspiciousFaceImage(matchScore, livenessScore, warnings);
if (isSuspicious) {
  // رفض التحقق
}
```

## تدفق التحقق المحسّن

```
1. إرسال رمز OTP
   ↓
2. التحقق من رمز OTP
   ├─ نجح → إرسال إشعار نجاح
   └─ فشل → إرسال إشعار فشل

3. رفع بطاقة الهوية
   ├─ استخراج البيانات بـ OCR
   ├─ التحقق من جودة البيانات
   ├─ فحص الهويات المكررة
   ├─ تقييم مخاطر الاحتيال
   ├─ نجح → إرسال إشعار نجاح
   └─ فشل → إرسال إشعار فشل

4. رفع السيلفي
   ├─ التحقق من جودة الصورة
   ├─ نجح → إرسال إشعار نجاح
   └─ فشل → إرسال إشعار فشل

5. مطابقة الوجه
   ├─ مقارنة الوجوه
   ├─ كشف الحياة
   ├─ فحص الصور المريبة
   ├─ حساب درجة المخاطر
   ├─ نجح → تحديث مستوى التحقق إلى 3
   ├─ مريب → وضع تحت المراجعة
   └─ فشل → إرسال إشعار فشل

6. مراجعة يدوية (للحالات المريبة)
   ├─ موافقة → إرسال إشعار موافقة
   └─ رفض → إرسال إشعار رفض
```

## معايير الموافقة النهائية

### المتطلبات الإلزامية:
- ✓ التحقق من الهاتف (المستوى 1)
- ✓ رفع بطاقة الهوية (المستوى 2)
- ✓ رفع السيلفي (المستوى 2)
- ✓ مطابقة الوجه >= 90% (المستوى 3)
- ✓ درجة الثقة >= 70%
- ✓ درجة كشف الحياة >= 80% (إن وجدت)
- ✓ درجة مخاطر الاحتيال < 30

### المتطلبات الإضافية:
- ✓ عدم وجود هوية مكررة
- ✓ عدم وجود نشاط مريب
- ✓ عدم وجود علامات تحرير أو تزييف

## الأمان والخصوصية

### تشفير البيانات:
- رقم الهوية الوطنية: **AES-256-GCM**
- بيانات الدفع: **AES-256-GCM**
- الرسائل الحساسة: **AES-256-GCM**

### التجزئة:
- رقم الهوية: **SHA-256** (لمنع التكرار)

### السجلات:
- جميع محاولات التحقق مسجلة
- البيانات الحساسة مخفية في السجلات
- تتبع IP وعامل المستخدم
