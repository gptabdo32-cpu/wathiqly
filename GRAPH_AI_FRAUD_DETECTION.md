# نظام كشف الاحتيال عبر الشبكات المرتبطة (Graph AI Fraud Detection)

يوفر هذا النظام طبقة أمان متقدمة تعتمد على **الذكاء الاصطناعي لتحليل الرسوم البيانية (Graph AI)** لكشف شبكات الاحتيال المنظمة والأنشطة المشبوهة التي يصعب كشفها بالوسائل التقليدية.

## 🚀 المميزات الرئيسية

*   **تحليل الروابط المخفية**: ربط الحسابات بناءً على (نفس الجهاز، نفس الـ IP، نفس نمط المعاملات).
*   **كشف هجمات Sybil**: التعرف التلقائي على "عصابات الاحتيال" التي تدير مئات الحسابات من مصدر واحد.
*   **تقييم المخاطر بالذكاء الاصطناعي**: استخدام نماذج GPT-4 المتقدمة لتحليل تعقيد الشبكات وإعطاء درجة خطورة (Risk Score).
*   **محرك الرسوم البيانية (Graph Engine)**: تتبع العلاقات بعمق متعدد (Multi-depth) لكشف المستفيدين النهائيين من عمليات الاحتيال.

## 🛠 التقنيات المستخدمة

| التقنية | الاستخدام |
| :--- | :--- |
| **Drizzle ORM** | إدارة مخطط البيانات والعلاقات المعقدة |
| **Graph AI (GPT-4.1)** | تحليل الأنماط واتخاذ قرارات ذكية بشأن مستوى الخطورة |
| **Relational Graphing** | تمثيل الكيانات (مستخدمين، أجهزة، محافظ) كعقد وروابط |
| **tRPC** | توفير واجهات برمجية آمنة للمسؤولين لمراقبة الشبكات |

## 🏗 المعمارية التقنية

يعتمد النظام على أربعة مكونات أساسية:

1.  **طبقة تتبع الكيانات (Nodes)**: يتم تسجيل كل مستخدم، جهاز، وعنوان IP كعقدة فريدة في النظام.
2.  **طبقة الروابط (Edges)**: يتم تسجيل التفاعلات (تسجيل دخول، تحويل مالي، مشاركة جهاز) كروابط بين العقد مع أوزان (Weights) تزداد بتكرار النشاط.
3.  **محرك الكشف (Detection Engine)**: خوارزميات تبحث عن التجمعات (Clusters) المشبوهة حيث يتشارك عدد كبير من المستخدمين في موارد محدودة.
4.  **محلل الذكاء الاصطناعي (AI Analyst)**: يقوم بمراجعة الشبكات المكتشفة وفهم السياق البشري وراءها لتجنب الإنذارات الكاذبة.

## 🛡 كيفية الاستخدام (للمسؤولين)

يمكن للمسؤولين الوصول إلى لوحة تحكم الاحتيال عبر الواجهات التالية:
*   `fraud.getUserNetwork`: عرض الشبكة المرتبطة بمستخدم معين.
*   `fraud.getDetectedClusters`: عرض تجمعات الاحتيال المكتشفة حالياً.
*   `fraud.triggerFraudScan`: بدء فحص شامل للنظام لكشف الشبكات الجديدة.
