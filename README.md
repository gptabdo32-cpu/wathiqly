# وثّقلي (Wathiqly) - منصة الوساطة المالية الآمنة

منصة وساطة مالية آمنة (Escrow) متطورة موجهة للسوق الليبي، تهدف إلى توفير أعلى مستويات الحماية والثقة في المعاملات التجارية الرقمية والفيزيائية.

## 🚀 نظرة عامة على المشروع

يهدف مشروع "وثّقلي" إلى إنشاء بيئة تجارية موثوقة تعمل كوسيط ذكي بين البائعين والمشترين. تعتمد المنصة على نموذج حجز الأموال (Escrow) لضمان حقوق الطرفين، حيث لا يتم تحرير المبالغ للبائع إلا بعد تأكيد المشتري لاستلام المنتج أو الخدمة وفق المواصفات المتفق عليها.

## 🛠 التقنيات المستخدمة (Modern Tech Stack)

تم بناء المشروع باستخدام أحدث التقنيات لضمان الأداء العالي والأمان الفائق:

| الطبقة | التقنية | المميزات |
| :--- | :--- | :--- |
| **الواجهة الأمامية** | React 19, TypeScript, Vite, TailwindCSS | واجهة مستخدم تفاعلية، سريعة، ومتجاوبة بالكامل. |
| **الواجهة الخلفية** | Node.js 22, Express, tRPC | هندسة Type-safe بالكامل، أداء عالٍ، وأمان مدمج. |
| **قاعدة البيانات** | MySQL, Drizzle ORM | هيكل بيانات موحد، استعلامات سريعة، وإدارة احترافية للهجرات. |
| **الأمان** | AES-256-GCM, Helmet, Rate Limiting | تشفير عسكري للبيانات الحساسة وحماية ضد الهجمات الشائعة. |
| **البنية التحتية** | Docker, GitHub Actions (CI/CD) | بيئة تشغيل معزولة وأتمتة كاملة لعمليات الفحص والنشر. |

## 🛡 التحسينات الهندسية والأمنية المطبقة (Engineering Excellence)

تمت إعادة هيكلة المشروع بالكامل ليتوافق مع أعلى المعايير العالمية:

### 1. الأمان المتقدم (Security Hardening)
*   **توحيد الطبقة الأمنية**: دمج جميع آليات الحماية في `core/security.ts` بما يشمل (CORS, Helmet, Rate Limiting, Sanitization).
*   **تشفير البيانات**: تطبيق نظام تشفير متطور للبيانات المالية وتفاصيل الهوية قبل تخزينها.
*   **أمان العقود الذكية**: تحديث `SmartEscrow.sol` ليشمل حماية ضد هجمات Reentrancy، ونظام إيقاف الطوارئ (Pausable)، وتحسين منطق الرسوم.
*   **تأمين الملفات**: إضافة قيود صارمة على أنواع وأحجام الملفات المرفوعة لمنع رفع ملفات ضارة.

### 2. جودة الكود وإعادة الهيكلة (Refactoring)
*   **توحيد قاعدة البيانات**: دمج أكثر من 6 ملفات Schema مكررة في ملف `drizzle/schema.ts` واحد لضمان "مصدر واحد للحقيقة" (Single Source of Truth).
*   **إزالة التكرار (DRY)**: دمج ملفات الـ API والـ Middlewares المتكررة وتوحيد منطق العمل (Business Logic) في السيرفر.
*   **التعامل مع العمليات المالية**: استخدام مكتبة `decimal.js` لضمان دقة الحسابات المالية وتجنب أخطاء الفواصل العائمة.

### 3. المعايير الهندسية (DevOps & Standards)
*   **Dockerization**: توفير `Dockerfile` و `.dockerignore` لضمان سهولة النشر والتشغيل في أي بيئة.
*   **CI/CD Pipeline**: إعداد سير عمل آلي عبر GitHub Actions يقوم بفحص الكود (Linting) وبناء المشروع عند كل عملية دفع.
*   **سجل التدقيق (Audit Logs)**: تفعيل نظام تتبع كامل لجميع العمليات الحساسة (تغيير الملف الشخصي، العمليات المالية، النزاعات) لضمان الشفافية.

## 🌟 الميزات الرئيسية
*   **الضمان الذكي (Smart Escrow)**: دعم المراحل (Milestones) والتحقق الآلي.
*   **تكامل IoT**: دعم أجهزة التتبع والحساسات لضمان سلامة الشحنات الفيزيائية.
*   **مركز النزاعات**: نظام متكامل لإدارة الخلافات مع دعم رفع الأدلة والدردشة الموثقة.
*   **نظام الهوية (KYC)**: إجراءات صارمة للتحقق من هوية المستخدمين لزيادة الموثوقية.
*   **الروابط الموقوتة**: إمكانية إنشاء روابط دفع تنتهي صلاحيتها آلياً.

## 📦 التشغيل والتطوير

### المتطلبات
*   Node.js 22+
*   pnpm 9+
*   MySQL 8+

### التثبيت
```bash
pnpm install
pnpm run dev
```

### باستخدام Docker
```bash
docker build -t wathiqly .
docker run -p 3000:3000 wathiqly
```
