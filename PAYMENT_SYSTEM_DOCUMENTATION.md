# نظام الدفع المتقدم - وثّقلي (Wathiqly)

## نظرة عامة

تم تطوير نظام دفع متقدم وشامل يدعم جميع وسائل الدفع الليبية الحالية (مارس 2026) مع ضمان الشفافية الكاملة والأمان العالي.

---

## 🎯 الميزات الرئيسية

### 1. وسائل الدفع المدعومة

| الوسيلة | الرمز | العمولة | الحالة | الملاحظات |
| :--- | :--- | :--- | :--- | :--- |
| **رصيد الهاتف** | `phone_credit` | 30% | فعال | ليبيانا / المدار - تحويل 100 إلى 70 |
| **بطاقة تعبئة** | `topup_card` | 1% | فعال | كروت تعبئة المنصة الخاصة |
| **سداد (SADAD)** | `sadad` | 1% | فعال | محفظة سداد الإلكترونية من المدار |
| **تداول (Tadawul)** | `tadawul` | 1% | فعال | نقاط البيع والدفع الإلكتروني |
| **إدفعلي (Edfaali)** | `edfaali` | 1% | فعال | مصرف التجارة والتنمية |
| **تحويل بنكي** | `bank_transfer` | 2% | فعال | جميع المصارف الليبية (متوسط 2%) |
| **كاش / فرع** | `cash` | 0% | فعال | الإيداع المباشر في الفروع |

### 2. خوارزميات العمولات

#### أ) حساب العمولة التلقائي

```typescript
const commission = amount * COMMISSIONS[paymentMethod];
const convertedAmount = amount - commission;
```

#### ب) أمثلة حقيقية

**مثال 1: رصيد هاتف (30% عمولة)**
- المبلغ المدفوع: 100 د.ل
- العمولة: 30 د.ل
- الرصيد النهائي: 70 د.ل

**مثال 2: تحويل بنكي (2% عمولة)**
- المبلغ المدفوع: 100 د.ل
- العمولة: 2 د.ل
- الرصيد النهائي: 98 د.ل

**مثال 3: سداد (1% عمولة)**
- المبلغ المدفوع: 50 د.ل
- العمولة: 0.5 د.ل
- الرصيد النهائي: 49.5 د.ل

### 3. الشفافية والأمان

#### عرض العمولات قبل التأكيد
- يتم عرض المبلغ الأصلي والعمولة والصافي قبل أي تأكيد
- لا توجد رسوم مخفية أو مفاجآت
- المستخدم يرى بوضوح ما سيحصل عليه

#### التشفير الكامل
- جميع بيانات الدفع مشفرة باستخدام AES-256
- بيانات الحساب البنكي مخزنة بشكل آمن
- سجل تدقيق (Audit Log) لكل عملية

#### عدم القابلية للتراجع
- بمجرد تأكيد عملية تحويل الرصيد، لا يمكن التراجع
- تحذير واضح قبل التأكيد النهائي
- سجل دائم لكل عملية

- تأكيد نجاح العملية
- رقم المرجع للعملية

## 🔧 الخلفية (Backend)

### قاعدة البيانات

#### جدول `depositRequests`

```sql
CREATE TABLE depositRequests (
  id INT PRIMARY KEY AUTO_INCREMENT,
  userId INT NOT NULL,
  amount DECIMAL(15, 2) NOT NULL,
  convertedAmount DECIMAL(15, 2) NOT NULL,
  paymentMethod ENUM(...) NOT NULL,
  paymentDetails TEXT NOT NULL, -- مشفر
  status ENUM('pending', 'completed', 'failed', 'cancelled'),
  adminNotes TEXT,
  createdAt TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updatedAt TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

#### جدول `wallets` (محدث)

- `balance`: الرصيد المتاح
- `pendingBalance`: الرصيد المعلق
- `totalEarned`: إجمالي الأرباح
- `totalWithdrawn`: إجمالي السحب

#### جدول `transactions` (محدث)

- نوع جديد: `deposit`
- حقل `reference`: معرّف العملية

### TRPC Router

#### `wallet.requestDeposit`

```typescript
requestDeposit: protectedProcedure
  .input({
    amount: string,
    convertedAmount: string,
    paymentMethod: PaymentMethod,
    paymentDetails: Record<string, any>
  })
  .mutation(async ({ ctx, input }) => {
    // 1. إنشاء طلب إيداع
    // 2. إنشاء سجل معاملة
    // 3. تسجيل الحدث في السجل
    // 4. إرجاع معرّف الطلب
  })
```

### التدفق الكامل

```
1. المستخدم يختار الوسيلة والمبلغ
   ↓
2. الواجهة تحسب العمولة والصافي
   ↓
3. المستخدم يؤكد العملية
   ↓
4. طلب يُرسل إلى Backend
   ↓
5. Backend ينشئ:
   - depositRequest (حالة: pending)
   - transaction (نوع: deposit)
   - auditLog (لتتبع الحدث)
   ↓
6. واجهة تعرض رسالة النجاح
   ↓
7. (في المستقبل) webhook يؤكد الدفع
   ↓
8. Backend يحدث:
   - depositRequest (حالة: completed)
   - wallet (يضيف الرصيد)
   - transaction (حالة: completed)
```

## 📞 الدعم الفني

### الأسئلة الشائعة

**س: هل يمكن استرجاع المبلغ بعد التحويل؟**
ج: لا، العملية غير قابلة للتراجع. لكن يمكنك التواصل مع الدعم في حالة الخطأ.

**س: كم يستغرق وصول الرصيد؟**
ج: يعتمد على الوسيلة:
- سداد/تداول/إدفعلي: فوري
- تحويل بنكي: 1-3 أيام عمل
- كاش: بعد التحقق (عادة ساعات)

**س: هل هناك حد أقصى للإيداع؟**
ج: نعم، حسب السياسة المحددة من قبل الإدارة.
