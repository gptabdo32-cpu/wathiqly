# دليل ميزة الوسيط الحي (Live Mediator Feature)

## نظرة عامة

ميزة الوسيط الحي توفر نظاماً متكاملاً لحل النزاعات داخل غرفة الدردشة نفسها، مما يعزز الثقة والأمان بين المشترين والبائعين على منصة Wathiqly.

## المميزات الرئيسية

### 1. طلب الوسيط
- **الزر:** يظهر زر "طلب وسيط (+10 دينار)" في أسفل الدردشة
- **الرسم:** 10 دينار فقط لطلب الوسيط
- **الوقت:** سيتم تعيين وسيط في غضون ساعتين
- **الحالة:** يتم عرض حالة الطلب (قيد الانتظار، مقبول، نشط، تم الحل)

### 2. الوسيط الحي في الدردشة
- **الدخول:** بعد تعيين الوسيط، يدخل الدردشة كطرف ثالث
- **الاسم:** يظهر باسم "وسيط وثقلي"
- **الرسائل:** جميع رسائل الوسيط موثقة ولا يمكن حذفها
- **الأدوات:** للوسيط صلاحيات خاصة (تجميد الصفقة، اتخاذ قرار)

### 3. المحادثات الخاصة
- **الخصوصية:** يمكن للوسيط إجراء محادثات خاصة مع كل طرف على حدة
- **الحماية:** المحادثات الخاصة لا تظهر للطرف الآخر
- **المرونة:** الوسيط يقرر متى يشارك المعلومات مع الطرفين

### 4. قرارات الوسيط
- **أنواع القرارات:**
  - تحرير المبلغ كاملاً للبائع
  - استرجاع المبلغ كاملاً للمشتري
  - تقسيم المبلغ بين الطرفين
  - توزيع مخصص

- **التوثيق:** كل قرار يتم توثيقه مع السبب والأدلة
- **الاستئناف:** يمكن استئناف القرار في حالات معينة

### 5. تجميد الصفقة
- **الصلاحية:** للوسيط فقط
- **السبب:** يتم تسجيل سبب التجميد
- **الإفراج:** يمكن إلغاء التجميد بعد حل المشكلة

## البنية التقنية

### جداول قاعدة البيانات الجديدة

#### 1. `mediator_requests`
يخزن طلبات الوسيط وحالتها:
```sql
- id: معرف الطلب
- conversationId: معرف الدردشة
- escrowId: معرف الصفقة
- requestedBy: من طلب الوسيط
- mediatorId: معرف الوسيط المعين
- status: حالة الطلب (pending, accepted, active, resolved, cancelled)
- fee: رسم الوسيط (10 دينار)
- feeTransactionId: معرف معاملة الرسم
- reason: سبب طلب الوسيط
- requestedAt: وقت الطلب
- acceptedAt: وقت قبول الوسيط
- resolvedAt: وقت حل النزاع
- resolution: تفاصيل الحل
```

#### 2. `mediator_messages`
رسائل الوسيط في الدردشة العامة:
```sql
- id: معرف الرسالة
- mediatorRequestId: معرف طلب الوسيط
- conversationId: معرف الدردشة
- senderId: معرف الوسيط
- messageType: نوع الرسالة (text, decision, freeze, unfreeze, evidence_request)
- content: محتوى الرسالة
- isSystemMessage: هل رسالة نظام تلقائية
- canBeDeleted: هل يمكن حذف الرسالة (دائماً false للوسيط)
- createdAt: وقت الإنشاء
```

#### 3. `mediator_private_chats`
المحادثات الخاصة بين الوسيط والطرفين:
```sql
- id: معرف المحادثة الخاصة
- mediatorRequestId: معرف طلب الوسيط
- mediatorId: معرف الوسيط
- userId: معرف المستخدم (بائع أو مشتري)
- isActive: هل المحادثة نشطة
- createdAt: وقت الإنشاء
```

#### 4. `mediator_decisions`
القرارات النهائية للوسيط:
```sql
- id: معرف القرار
- mediatorRequestId: معرف طلب الوسيط
- escrowId: معرف الصفقة
- mediatorId: معرف الوسيط
- decisionType: نوع القرار
- buyerAmount: المبلغ للمشتري
- sellerAmount: المبلغ للبائع
- reason: سبب القرار
- evidence: الأدلة المرفقة
- isAppealed: هل تم استئناف القرار
- appealReason: سبب الاستئناف
- finalDecision: هل هذا القرار نهائي
```

#### 5. `mediator_freeze_logs`
سجل تجميد الصفقات:
```sql
- id: معرف السجل
- mediatorRequestId: معرف طلب الوسيط
- escrowId: معرف الصفقة
- mediatorId: معرف الوسيط
- frozenAt: وقت التجميد
- unfrozenAt: وقت الإفراج
- reason: سبب التجميد
```

### المسارات (Routes) في tRPC

#### `mediator.requestMediator`
**الوصف:** طلب وسيط لنزاع معين
**المدخلات:**
- `conversationId`: معرف الدردشة
- `reason`: سبب طلب الوسيط (10-500 حرف)

**المخرجات:**
```javascript
{
  success: boolean,
  mediatorRequestId: number,
  message: string
}
```

**الخطوات:**
1. التحقق من أن المستخدم طرف في الدردشة
2. التحقق من عدم وجود طلب وسيط نشط
3. التحقق من رصيد المستخدم (يجب أن يكون 10 دينار على الأقل)
4. خصم الرسم من محفظة المستخدم
5. إنشاء طلب وسيط جديد
6. إضافة رسالة نظام إلى الدردشة

#### `mediator.getMediatorRequest`
**الوصف:** الحصول على تفاصيل طلب وسيط
**المدخلات:**
- `requestId`: معرف الطلب

**المخرجات:** كائن طلب الوسيط كاملاً

#### `mediator.getMediatorMessages`
**الوصف:** الحصول على رسائل الوسيط في دردشة معينة
**المدخلات:**
- `conversationId`: معرف الدردشة
- `limit`: عدد الرسائل (افتراضي: 50)
- `offset`: الإزاحة (افتراضي: 0)

**المخرجات:** مصفوفة من رسائل الوسيط

#### `mediator.sendMediatorMessage`
**الوصف:** إرسال رسالة من الوسيط إلى الدردشة
**المدخلات:**
- `mediatorRequestId`: معرف طلب الوسيط
- `conversationId`: معرف الدردشة
- `content`: محتوى الرسالة
- `messageType`: نوع الرسالة (text, decision, freeze, etc.)

**المخرجات:**
```javascript
{
  success: boolean,
  messageId: number
}
```

#### `mediator.makeMediatorDecision`
**الوصف:** اتخاذ قرار نهائي بشأن النزاع
**المدخلات:**
- `mediatorRequestId`: معرف طلب الوسيط
- `decisionType`: نوع القرار
- `buyerAmount`: المبلغ للمشتري (اختياري)
- `sellerAmount`: المبلغ للبائع (اختياري)
- `reason`: سبب القرار
- `evidence`: الأدلة (اختياري)

**المخرجات:**
```javascript
{
  success: boolean,
  decisionId: number
}
```

#### `mediator.freezeEscrow`
**الوصف:** تجميد صفقة معينة (للوسيط فقط)
**المدخلات:**
- `mediatorRequestId`: معرف طلب الوسيط
- `reason`: سبب التجميد

**المخرجات:**
```javascript
{
  success: boolean
}
```

## المكونات (Components)

### 1. `RequestMediatorButton`
زر طلب الوسيط مع نافذة حوار احترافية.

**الخصائص:**
- `conversationId`: معرف الدردشة
- `onMediatorRequested`: دالة callback عند طلب الوسيط

**الميزات:**
- تصميم احترافي بألوان برتقالية
- نافذة حوار لإدخال السبب
- تأكيد قبل الدفع
- عرض الرسم والوقت

### 2. `MediatorPanel`
لوحة عرض الوسيط داخل الدردشة.

**الخصائص:**
- `mediatorRequestId`: معرف طلب الوسيط
- `conversationId`: معرف الدردشة
- `mediatorId`: معرف الوسيط
- `mediatorName`: اسم الوسيط
- `status`: حالة الطلب

**الميزات:**
- عرض حالة الطلب
- رسائل الوسيط
- محادثات خاصة
- أدوات الوسيط (تجميد، قرار)

### 3. `ChatBoxWithMediator`
مكون الدردشة المحسّن مع دعم الوسيط.

**الخصائص:**
- `conversationId`: معرف الدردشة
- `otherUserName`: اسم المستخدم الآخر
- `otherUserId`: معرف المستخدم الآخر
- `escrowId`: معرف الصفقة

**الميزات:**
- جميع ميزات الدردشة الأصلية
- زر طلب الوسيط
- لوحة الوسيط
- تحديث تلقائي لحالة الوسيط

## تدفق العمل (Workflow)

### 1. طلب الوسيط
```
المستخدم يضغط على "طلب وسيط"
    ↓
يدخل السبب ويؤكد
    ↓
يتم خصم 10 دينار من رصيده
    ↓
يتم إنشاء طلب وسيط بحالة "pending"
    ↓
رسالة نظام تظهر في الدردشة
```

### 2. تعيين الوسيط
```
المسؤول يعين وسيط متاح
    ↓
حالة الطلب تتغير إلى "accepted"
    ↓
الوسيط يتلقى إشعار بالطلب
    ↓
الوسيط يدخل الدردشة
```

### 3. الوسيط يعمل على النزاع
```
الوسيط يقرأ الرسائل السابقة
    ↓
قد يطلب أدلة إضافية
    ↓
قد يجري محادثات خاصة مع الطرفين
    ↓
يتخذ قرار نهائي
```

### 4. تنفيذ القرار
```
الوسيط يرسل القرار
    ↓
يتم تحديث حالة الصفقة
    ↓
يتم تحويل الأموال حسب القرار
    ↓
يتم إغلاق النزاع
```

## الأمان والحماية

### 1. عدم القدرة على الحذف
- رسائل الوسيط لا يمكن حذفها أبداً
- جميع الرسائل موثقة في قاعدة البيانات
- سجل كامل للعمليات

### 2. التحقق من الصلاحيات
- فقط الوسيط المعين يمكنه إرسال رسائل الوسيط
- فقط الأطراف المعنيين يمكنهم رؤية الدردشة
- المحادثات الخاصة محمية بشكل كامل

### 3. الرسوم والمحاسبة
- الرسم يتم خصمه فوراً
- يتم تسجيل معاملة للرسم
- سجل كامل للمدفوعات


## الأسئلة الشائعة

### س: هل يمكن للمستخدم طلب وسيط أكثر من مرة؟
**ج:** لا، يمكن طلب وسيط واحد فقط لكل دردشة في المرة الواحدة.

### س: ماذا يحدث إذا لم يكن لدى المستخدم 10 دينار؟
**ج:** سيتم رفض الطلب ويظهر رسالة خطأ.

### س: هل يمكن استرجاع رسم الوسيط؟
**ج:** لا، الرسم غير قابل للاسترجاع.

### س: هل الوسيط يحصل على الرسم؟
**ج:** يمكن تحديد نسبة توزيع الرسم بين الوسيط والمنصة.

### س: كم عدد الوسطاء المتاحين؟
**ج:** يعتمد على عدد المستخدمين المسجلين كوسطاء.

## الدعم والمساعدة

للمزيد من المعلومات أو الإبلاغ عن مشاكل، يرجى التواصل مع فريق الدعم.
