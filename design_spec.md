# مواصفات تصميم محفظة "وثّقلي" المتكاملة للهوية والمدفوعات (Wathiqly ID & Pay Wallet)

## 1. نظرة عامة
تحويل منصة "وثّقلي" من نظام وساطة (Escrow) إلى نظام بيئي متكامل للهوية الرقمية والمدفوعات في ليبيا، مع التركيز على الأمان والمعايير العالمية.

## 2. الميزات الرئيسية
### أ. مدفوعات P2P الفورية
*   **الوصف**: إرسال واستقبال الأموال بين المستخدمين باستخدام رقم الهاتف الموثق.
*   **المعايير**: تتبع معايير PCI-DSS في معالجة البيانات المالية.
*   **الأمان**: التحقق من الهوية (KYC) قبل تفعيل ميزة الإرسال، مع سجل تدقيق (Audit Log) كامل.

### ب. دفع الفواتير والخدمات
*   **التكامل**: واجهات برمجية (APIs) لخدمات "ليبيانا" و"المدار" والكهرباء.
*   **العملية**: خصم مباشر من محفظة "وثّقلي" مع إصدار إيصال رقمي موثق.

### ج. الهوية الرقمية الموثوقة (SSO)
*   **الوصف**: استخدام حساب "وثّقلي" كبوابة دخول موحدة للخدمات الخارجية.
*   **المعايير**: تطبيق بروتوكول **OAuth 2.0 / OpenID Connect**.
*   **الأمان**: توفير رموز وصول (Access Tokens) محدودة الصلاحية ومشفرة.

## 3. الهندسة التقنية (Architecture)
### أ. قاعدة البيانات (Database Schema)
*   `wallet_transfers`: لتتبع تحويلات P2P.
*   `bill_payments`: لتتبع مدفوعات الفواتير.
*   `oauth_clients` & `oauth_tokens`: لإدارة الهوية الرقمية الموحدة.

### ب. طبقة الأمان (Security Layer)
*   **تشفير البيانات**: استخدام AES-256 للبيانات الحساسة (مثل أرقام الهوية).
*   **التحقق الثنائي (2FA)**: إلزامي للعمليات المالية الكبيرة.
*   **Rate Limiting**: حماية الـ APIs من هجمات التخمين والإغراق.

## 4. واجهة المستخدم (UI/UX)
*   لوحة تحكم موحدة للمحفظة تعرض الرصيد، التحويلات، والفواتير.
*   واجهة بسيطة لإدخال رقم الهاتف والمبلغ للتحويل السريع.
*   قسم خاص بإدارة التطبيقات المرتبطة بالهوية الرقمية.

## 5. خطة التنفيذ
1.  تحديث مخطط قاعدة البيانات (Drizzle Schema).
2.  تطوير الـ API الخلفي (TRPC Routers).
3.  بناء واجهات المستخدم (React Components).
4.  اختبار الأمان والتكامل.
